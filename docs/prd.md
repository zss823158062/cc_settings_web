# AI 工具配置生成器 - 产品需求文档

## 1. 项目概述

### 1.1 背景与目标

**项目背景**：
开发者在使用 Claude Code、Codex、Gemini CLI 等 AI 编程工具时，需要手动编写和维护各自的配置文件（JSON/TOML 格式）。配置项繁多且格式各异，容易出错，缺乏可视化的配置管理工具。

**核心目标**：
1. 提供可视化的配置表单，降低配置文件编写门槛
2. 支持多种 AI 工具的配置生成和导出（Codex/Claude Code/Gemini CLI）
3. 实现配置文件的导入、编辑、预览和导出完整流程

**成功指标**：
- 用户可在 5 分钟内完成一个工具的完整配置
- 生成的配置文件格式正确率 100%
- 支持离线使用（纯前端实现）

### 1.2 目标用户

**用户画像**：
- 使用 AI 编程工具的开发者（初级到高级）
- 需要管理多个 AI 工具配置的团队
- 希望快速切换不同配置场景的用户（开发/生产/安全等）

**使用场景**：
- 场景 1：新用户首次配置 Claude Code，不熟悉 JSON 格式
- 场景 2：团队需要统一配置模板，快速分发给成员
- 场景 3：开发者需要在不同项目间切换配置（如开发环境 vs 生产环境）
- 场景 4：用户想导入现有配置文件进行微调

### 1.3 技术约束

**技术栈要求**：
- 前端框架：React 18 + TypeScript
- 构建工具：Vite
- 样式方案：Tailwind CSS
- 表单管理：React Hook Form + Zod（表单验证）
- 配置解析：@iarna/toml（TOML 解析）、原生 JSON API

**部署环境**：
- 部署平台：Cloudflare Pages
- 运行环境：纯静态站点，无后端依赖
- 所有处理在浏览器端完成（文件读写、格式转换）

**性能要求**：
- 首屏加载时间 < 2s（3G 网络）
- 表单交互响应 < 100ms
- 配置文件生成 < 500ms

**兼容性要求**：
- 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 响应式设计：支持桌面端（1280px+）和平板端（768px+）
- 移动端为次要支持（可用但非最佳体验）

## 3. 非功能需求

### 3.1 性能要求

- 首屏加载时间 < 2s（3G 网络，未缓存）
- 首屏加载时间 < 500ms（已缓存）
- 表单输入响应时间 < 100ms
- 配置文件生成时间 < 500ms
- 文件导入解析时间 < 1s（< 1MB 文件）
- 页面切换动画流畅（60fps）

### 3.2 安全要求

- 所有数据处理在浏览器本地完成，不上传到服务器
- API Key 等敏感信息不存储到 localStorage（仅在内存中）
- 导入文件时验证文件类型和大小（< 5MB）
- 防止 XSS 攻击：用户输入内容转义后显示
- 使用 HTTPS 部署（Cloudflare Pages 默认）

### 3.3 可用性要求

- 浏览器兼容：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 响应式设计：支持 1280px+ 桌面端、768px+ 平板端
- 移动端（< 768px）可用但非最佳体验（表单和预览垂直排列）
- 键盘导航：支持 Tab 键切换字段、Enter 键提交
- 无障碍访问：表单字段有 label、按钮有 aria-label
- 错误提示清晰：避免技术术语，使用用户友好的语言

## 4. 数据模型（初步）

### 4.1 核心实体

**ConfigForm（配置表单）**：
- tool: string（工具类型：codex / claude / gemini）
- fields: Field[]（表单字段列表）
- values: Record<string, any>（表单值）
- errors: Record<string, string>（验证错误）

**Field（表单字段）**：
- name: string（字段名）
- label: string（显示标签）
- type: string（字段类型：text / number / boolean / array / object）
- required: boolean（是否必填）
- defaultValue: any（默认值）
- validation: ValidationRule[]（验证规则）

**Template（配置模板）**：
- id: string（模板 ID）
- name: string（模板名称）
- description: string（模板描述）
- tool: string（适用工具）
- values: Record<string, any>（预设值）

### 4.2 API 概要

本项目为纯前端应用，无后端 API。所有数据处理通过浏览器 API 完成：

**文件操作**：
- 导入：File API（FileReader）
- 导出：Blob API + URL.createObjectURL
- 复制：Clipboard API（navigator.clipboard.writeText）

**数据存储**：
- 主题偏好：localStorage.setItem('theme', 'dark')
- 表单数据：仅在内存中（React state），不持久化

**格式转换**：
- JSON 解析：JSON.parse() / JSON.stringify()
- TOML 解析：@iarna/toml 库（parse / stringify）

### 4.3 模块间数据流概要

```
用户操作 → 表单组件 → React Hook Form → Zod 验证 → 状态更新 → 预览组件
                                                    ↓
                                              导出/复制功能
```

**关键数据转换点**：
1. 用户输入 → 表单值（onChange 事件）
2. 表单值 → 验证结果（Zod schema）
3. 表单值 → 配置文件字符串（JSON.stringify / TOML.stringify）
4. 配置文件字符串 → 下载文件（Blob）
5. 上传文件 → 表单值（FileReader + JSON.parse / TOML.parse）

### 4.4 跨模块共享概念列表

**工具类型（Tool）**：
- 命名：tool / toolType
- 格式：字符串枚举（'codex' | 'claude' | 'gemini'）
- 用途：标识当前配置的工具

**配置值（ConfigValues）**：
- 命名：configValues / formValues
- 格式：Record<string, any>（键值对对象）
- 用途：存储表单的所有字段值

**验证错误（ValidationErrors）**：
- 命名：errors / validationErrors
- 格式：Record<string, string>（字段名 → 错误信息）
- 用途：存储表单验证错误

**配置文件内容（ConfigContent）**：
- 命名：configContent / fileContent
- 格式：string（JSON 或 TOML 格式的字符串）
- 用途：预览区显示和导出的内容

**模板 ID（TemplateId）**：
- 命名：templateId
- 格式：string（如 'codex-dev' / 'claude-prod'）
- 用途：标识配置模板
