# AI Teams - 使用指南

## 快速开始

### 启动完整团队流水线

```
/ai-teams 做一个 Todo 应用，支持增删改查和分类管理
```

输入 `/ai-teams` + 需求描述，自动启动 6 阶段流水线：

1. 环境准备 → 2. 产品经理分析需求 → 3. 项目经理拆解任务 → 4. 搭建基础设施 → 5. 6人并行开发 → 6. 合并代码

每个阶段完成后会暂停等待你确认，你可以随时提出修改意见。

### 单独调用 Agent

如果只需要团队中的某个角色，在对话中直接说：

| 说法 | 效果 |
|------|------|
| 「使用产品经理分析需求：在线商城」 | 启动产品经理，迭代式讨论需求直到你确认 |
| 「让项目经理拆解这个任务」 | 启动项目经理，基于 PRD 拆解任务 |
| 「用前端开发完成这个页面」 | 启动前端开发 Agent 编码 |
| 「用后端开发实现这个接口」 | 启动后端开发 Agent 编码 |

## 团队成员

| 角色 | 模型 | 数量 | 职责 |
|------|------|------|------|
| 产品经理 | sonnet | 1 | 需求沟通 → 输出 `docs/prd.md` |
| 项目经理 | sonnet | 1 | 架构设计 → 任务拆解 → 输出 `docs/tasks.md` |
| 前端开发 | haiku | 2 | 在独立分支中完成前端编码 |
| 后端开发 | haiku | 4 | 在独立分支中完成后端编码 |

## 流水线阶段详解

### 阶段 0：环境准备
自动初始化 git 仓库、创建 docs 目录。

### 阶段 1：需求分析 ⏸️
产品经理分析需求并生成 PRD → 展示摘要 → **等待你确认**。

### 阶段 2：任务拆解 ⏸️
项目经理将 PRD 拆解为 6 个开发者的并行任务 → 展示分配 → **等待你确认**。

### 阶段 3：基础设施 ⏸️
项目经理搭建项目骨架（目录结构、依赖、共享类型等）→ 提交到 main → **等待你确认**。

### 阶段 4：并行开发 ⏸️
6 个开发者在独立的 git worktree 中同时编码 → 全部完成后报告状态 → **等待你确认**。

### 阶段 5：集成合并
按后端优先、前端其次的顺序合并分支 → 清理 worktree → 运行构建验证。

### 阶段 6：完成报告
输出团队执行摘要，包含统计信息和后续建议。

## 产出文件

| 文件 | 生成阶段 | 内容 |
|------|----------|------|
| `docs/prd.md` | 阶段 1 | 产品需求文档 |
| `docs/tasks.md` | 阶段 2 | 任务拆解与分配 |
| 项目代码 | 阶段 3-5 | 完整可运行的项目代码 |

## 建议

- 主 Agent（编排者）建议使用 **sonnet** 模型：`claude --model claude-sonnet-4-6`
- 需求描述越详细，产出质量越高
- 每个暂停点认真审阅，发现问题及时提出
- 单独调用产品经理可以反复沟通需求细节，不急着进入开发
